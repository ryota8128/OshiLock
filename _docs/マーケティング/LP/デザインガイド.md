# Oshilock LP デザインガイド

## デザイン戦略

**完全差別化路線。** 競合（Oshi UP!等）のポップ・カジュアル路線には寄せない。
OshiLockは「推し情報を逃さない実用ツール」として、信頼感・機能性で勝負する。

### ポジショニングの違い

| | Oshi UP! | Oshilock |
|---|---|---|
| 軸 | 推し活を楽しむ（記録・交流） | 推し情報を逃さない（整理・通知） |
| 訴求 | 楽しさ | 損失回避 |
| ターゲット | 学生〜20代前半、ライト層含む | 月1万円以上使う20-30代 |
| デザイン | ポップ、パステル、キャラ | ミニマル、warm、機能的 |

## ビジュアルスタイル

### やるべきこと

- **アプリUI実機モック**を主役にする（スクショを端末フレームに入れて表示）
- **余白を活かしたミニマル構成**（情報を詰め込まない）
- セクション間に十分なパディング
- 「情報が届く」体験をFVで可視化（AIが整理→通知のフロー図）

### やってはいけないこと

- マスコットキャラクター
- 絵文字・クラッカーの多用
- 「最強」「神」等の煽りコピー
- 背景いっぱいのイラストや装飾
- 情報の詰め込み

## コンポーネント

### CTA ボタン

- 背景: `#2B2A28`（ink）
- テキスト: `#FAF8F4`（paper）
- 角丸: 12px
- パディング: 16px 32px
- ホバー: opacity 0.9

### セクション

- 背景交互: `#FAF8F4` と `#F3EFE4` を交互に使用
- セクション間パディング: 80-120px（PC）/ 60-80px（SP）

### カード（イベントカード表示）

- 背景: `#FFFFFF`
- ボーダー: `rgba(0,0,0,0.08)`
- 角丸: 14px
- シャドウ: 軽め（0 2px 8px rgba(0,0,0,0.06)）
