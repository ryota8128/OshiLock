# OshiLock（オシロック）要件定義

## アプリ概要

**「推しの情報、ここだけ見ればOK」**

推しの情報が複数サイトに分散している問題を、運営による情報投入 + ファンのUGC投稿 + AIの自動整理で解決するファン向け情報コミュニティアプリ。

---

## 確定事項

### ターゲット・市場

- 推し活に月1万円以上使う「本気勢」
- 推し活人口: 1,384万人、市場: 4.1兆円
- 最初の対象グループはXの投票で決定する（1グループ限定で開始）

### MVP機能スコープ

**含める：**

- 推しグループ登録（2-3グループ、各100人招待）
- イベントカード閲覧
- 情報投稿（UGC）+ 運営によるシード投稿
- AIによる投稿の分類・マージ・重複排除
- コメント機能
- プッシュ通知（Expo Push Notifications）
- 認証（Apple Sign In + Google Sign In）

**含めない（後から追加）：**

- 課金機能（まず無料招待 → コミュニティが回ってから追加）
- 見逃しチェック（毎朝まとめ）
- カレンダー連携
- 推し活レポート
- Android版リリース

### 情報取得方式

- 運営がClaude Codeで情報を収集・投入（クローリングの詳細は運用時に決定）
- ユーザーもUGCとして情報を投稿可能
- AIが投稿を分類・マージしてイベントカード化

### 技術スタック

| コンポーネント | 技術 |
|--------------|------|
| アプリ | Expo（React Native） — iOS先行、Androidにも対応 |
| API | Hono on Vercel Functions |
| DB | DynamoDB（シングルテーブル設計、拡張性重視） |
| AI | プロバイダー・モデル切り替え可能な抽象層で実装 |
| 通知 | Expo Push Notifications |
| 認証 | Apple Sign In + Google Sign In |

### AI抽象層

- プロバイダー（Claude / OpenAI / Gemini 等）を環境変数で切り替え可能
- モデルも切り替え可能（例: claude-haiku-4-5, gpt-4o-mini, gemini-2.0-flash）
- コスト・品質を比較しながら最適なものを選定する

### 課金モデル（MVP後）

- 1グループ500円/月
- 7日間無料お試し
- 新コミュニティ開設時: 先着500人無料招待
