# OshiLock 機能詳細

## 画面構成

### タブ構成

```
[ホーム]  [タイムライン]  [気になる]  [マイページ]
+ 右下にFAB（投稿ボタン）
```

| タブ | 内容 |
|---|---|
| ホーム | 全グループ混合の未読件数 + 今日の予定。開いて3秒で状況把握 |
| タイムライン | カード一覧。デフォルト推しが初期表示、ピル型UIでグループ切り替え（「全て」も可）+ カテゴリフィルタ |
| 気になる | リマインド設定したカードの一覧。自分が追ってるものだけ確認 |
| マイページ | プロフィール、ランク表示、デフォルト推し設定、通知設定、アカウント管理 |

## イベントカード

### カテゴリと優先度

| カテゴリ | 色 | アイコン | 優先度 |
|---|---|---|---|
| イベント系（ライブ、握手会、チケット先行、グッズ発売） | ピンク | 🎫 | 最高 |
| メディア出演（未来） | パープル | 📺 | 高 |
| SNS投稿 | スカイブルー | 🔗 | 中 |
| その他ニュース / メディア出演（過去） | グレー | 📰 | 低 |

### カードに載る情報は何でもあり

- 日時があるもの（ライブ、チケット先行、メディア出演等）
- 日時がないもの（SNS投稿、ニュース等）
- 重複は弾く

### 時間経過でカードの役割が変わる

```
【放送前】
📺 ミュージックステーション出演
   7/5 22:00〜  あと3日
   ♡ 気になる 120
   💬 「楽しみ！」「何歌うかな」

      ↓ 放送後、自動で切り替わり

【放送後】
📺 ミュージックステーション出演
   7/5 放送済み
   ♡ 120
   💬 「最高だった！」「衣装かわいい→URL」
```

優先度の変化：
- 放送/開催前 → 高（未来のアクション）
- 終了後24時間 → 中（盛り上がり期間）
- 終了後24時間以降 → 低（通常のニュース扱い）

### カードの表示構造

```
┌─ カテゴリ色の縦ライン ─────────────────┐
│  🎫 イベント        あと2日            │
│                                       │
│  ワンマンライブ 東京ガーデンシアター      │
│  2026/7/20 18:00                      │
│                                       │
│  📡 最速: @sakura_fan                  │
│  ソース: 公式 ◎                        │
│                                       │
│  💬 12件   ♡ 気になる 48               │
└───────────────────────────────────────┘
```

- 左端にカテゴリ色の縦ライン（4px）でスクロール中でも瞬時にカテゴリ識別
- 締切24-48時間以内は赤い帯で「あと○日」カウントダウン表示
- 最速投稿者の名前をカードに表示

### 並び順ロジック

`緊急度 × 重要度 × 新着` のハイブリッド：

1. 締切が48時間以内のアクション系（チケット先行、グッズ発売）
2. 確定イベント（ライブ、握手会）の新着・更新
3. 未来のメディア出演（放送日が近い順）
4. SNS投稿、その他ニュース

## 情報投稿（UGC）

### 投稿方法

- 右下のFAB（+ボタン）から投稿
- テキストのみでもOK、URL付きも可
- 頻繁にやるものではないが、すぐ見つかる場所に配置

### 投稿の流れ

```
ユーザーが投稿（テキスト / URL）
  ↓
AIが自動で内容を解析・分類
  ↓
┌─ 既存カードに関連 → カードの裏側に情報として統合（確度UP）
│
└─ 新しい情報 → 新カード自動生成
```

- **ユーザー投稿はタイムラインに直接表示されない**（タイムラインの質を守る）
- AIが自動でカードに統合する（運営の手動承認なし）
- 運営もClaude Codeで情報を収集・投入（シード投稿）

### 情報の信頼性

| 条件 | 確度 |
|---|---|
| 公式ソースのURL付き | ◎（公式確認済み） |
| 複数人が同じ情報を投稿 | ○（確度UP） |
| 1人だけ + ソースなし | △（未確認情報ラベル） |

## 通知

### リマインド通知

- ユーザーが「気になる」ボタンを押すだけで自動設定（日時指定不要）
- カードのカテゴリに応じてAIが最適な通知タイミングを自動決定

| カテゴリ | 通知タイミング |
|---|---|
| チケット先行 | 開始前日 + 当日朝 |
| ライブ・握手会 | 前日 |
| メディア出演 | 1時間前 |
| グッズ発売 | 前日 |
| その他 | 当日朝 |

### 新着まとめ通知

- 1日1回「新着○件」で通知
- アプリを開くきっかけを作る

---

## 気になるボタン

- 「気になる」= リマインド設定を1ボタンに統合
- 押すと「気になる」タブに追加 + リマインド通知が設定される
- カードに「♡ 気になる 48」のように件数表示

---

## 情報カバー率バー

- ホームタブに表示
- ユーザー登録日以降に作成されたカードが対象（登録前のカードはカウント対象外）
- カードが画面に表示された時点で自動的に既読（明示的な操作不要）
- 「3件 未チェック / 9/12 チェック済」のような表示

---

## デマ通報

### 2段階の対応

1. 通報が一定数に達する → 「未確認情報」ラベルに格下げ
2. さらに通報が溜まる → 非表示化

- 誤通報対策として、いきなり非表示にはしない
- 通報の閾値は運用しながら調整

---

## 情報投稿の制限

| 制限 | 内容 |
|---|---|
| 投稿上限 | 1日3件まで |
| クールダウン | 投稿後5分間は次の投稿不可 |
| コメント | 制限なし |

## 最速投稿者 & ランク制度

### 最速投稿者

- カードの情報を1位・2位・3位で投稿したユーザーの名前をカードに表示
- 🥇 🥈 🥉 で表示

### ランク制度

- **評価軸:** 3位以内に入った回数
- **段階:** 5段階（上位X%で区切る）

| ランク | 条件 |
|---|---|
| レジェンド | 上位1% |
| スター | 上位5% |
| エース | 上位15% |
| レギュラー | 上位30% |
| ルーキー | 上位50% |
| ランクなし | それ以外（未投稿・投稿少） |

- ランクはマイページのプロフィールに表示
- **1年間はランク維持、年1回再判定**（一度上がったランクはすぐには下がらない）
- 将来の課金モデルでランクに応じた月額割引に活用予定（詳細はMVP後に決定）

## グループ構成

### MVP開始時

- 2-3グループで開始（対象はXの投票で決定）
- 各グループ100人招待
- 満員になったら追加枠を段階的に解放（「追加100枠解放！」→ Xで拡散、飢餓感でマーケティング）

### タイムラインでのグループ切り替え

```
🔘 乃木坂46 | 日向坂46 | 全て    ← ピル型UIで切り替え
─────────────────
フィルタ: イベント | メディア | …
─────────────────
カード一覧
```

- デフォルト推しをマイページで設定 → タイムライン初期表示
- 「全て」で全グループ混合表示も可能
- ホームタブは常に全グループ混合

## UGC投稿フォーム

### 入力項目

- テキスト: 必須（500文字以内）
- 参照URL: 任意・推奨（最大3件）
  - 「URLがあると確度が上がります」の推奨表示
  - URL付き → カードにソースリンク表示 + 確度◎
  - URLなし → 確度△（未確認情報）

### 投稿データの扱い

- AI判定 → カードに統合された時点で投稿データは破棄
- 投稿履歴画面は設けない
- 最速順位はカード側に記録

## 通報

### 通報カテゴリ

- デマ・虚偽情報
- スパム
- 不適切なコンテンツ（わいせつ・暴力等）
- 誹謗中傷・ハラスメント

### 対応フロー（既存のデマ対策と同じ2段階）

1. 通報が一定数 → 「未確認情報」ラベルに格下げ
2. さらに通報が溜まる → 非表示化

---

## 通知設定

- リマインド通知と新着まとめ通知を個別にON/OFF可能
- マイページの設定から切り替え

---

## 退会（アカウント削除）

- Apple審査要件で必須

| データ | 処理 |
|---|---|
| プロフィール情報 | 完全削除 |
| コメント | 匿名化（「退会済みユーザー」表示） |
| 最速投稿者表示 | 匿名化（順位繰り上げなし） |
| いいね | 紐付け削除、カウント数は維持 |
| 気になるリスト | 完全削除 |

- 退会後30日の猶予期間（その間は復旧可能）
- 猶予期間後に物理削除

## その他

- 利用規約・プライバシーポリシー: リリース前に用意（MVP実装後に作成）

## MVPに含めない機能

| 機能 | 理由 |
|---|---|
| 課金機能 | まず無料招待でコミュニティ検証 |
| 見逃しチェック（毎朝まとめ） | プッシュ通知で代替可能 |
| カレンダー連携 | MVP後に追加 |
| 推し活レポート | MVP後に追加 |
| Android版リリース | iOS先行、Expoで後にAndroid対応 |
| フォロー機能 | ランク制度で貢献者は可視化される。PMF後に再検討 |
| DM機能 | 情報がDMに閉じるリスク。カードコメントで十分 |
| カスタムアイコン | デフォルトアバターで開始、MVP後に追加 |
| ブロック/ミュート | 通報で対応。PMF後に再検討 |
| オフラインキャッシュ | MVP後に追加 |
